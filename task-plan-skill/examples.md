# 示例：从 PRD 到任务计划

## 输入

- 项目：蓝牙音响/功放控制 App
- 文档：PRD（含 UC-01~UC-09）
- 版本：V1.0

## 输出（节选）

```markdown
## 2. 里程碑
| 里程碑 | 时间窗口 | 目标产出 |
|---|---|---|
| M1 需求冻结 | W1 | PRD 冻结、接口草案 |
| M2 核心开发完成 | W2-W4 | 连接识别、核心控制完成 |
| M3 联调与测试 | W5 | 端到端联调通过 |
| M4 发布就绪 | W6 | 验收通过、发布包就绪 |

## 3. 任务分解（WBS）
| ID | 模块 | 任务 | 角色 | 依赖 | 优先级 | 交付物 |
|---|---|---|---|---|---|---|
| T-01 | 蓝牙连接 | 扫描/连接状态机实现 | 前端 | 无 | P0 | 连接模块代码 |
| T-02 | 设备识别 | 能力映射表与页面动态加载 | 前端 | T-01 | P0 | 映射配置与页面 |
| T-03 | 控制协议 | 音频参数指令封装 | 后端/中间层 | T-01 | P0 | 指令 SDK |
```
